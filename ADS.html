Original

Originally the WeightTracker application lacked important security measures and was not developed with a security mindset. Initially the program utilized the system's cache to store one user and lacked encryption of the username and password after it was stored in the database.

Expansion

I expanded the capabilities of the weight tracker app by creating encrypt/decrypt code that is modular and can be utilized by multiple functions within the code. By having this security measure in place, there is room for growth in the form of online database interactions, additional user information, and control of data presented to the user.

Forms of Security

AES Encryption

The user table currently utilizes AES encryption to encode and decode information while at rest. Utilizing AES encryption on this application helps to maintain its low memory usage and provides fast encryption and decryption times.

SQL Injection

In order to add another layer of security, all data entry points limit what the user can input. Dates are broken up by year, month, day, and are limited to numbers only and within a specific range. Username and passwords don't present much of a danger since the information utilized is encrypted prior to entering the database.
